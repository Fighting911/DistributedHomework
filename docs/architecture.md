# 系统架构设计

## 1. 整体架构

```
┌─────────────────────────────────────────────────────┐
│                      Client                          │
│              (Web Browser / App)                     │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP
┌──────────────────────▼──────────────────────────────┐
│                   API Gateway                        │
│            路由转发 / 鉴权 / 限流                     │
└──────┬───────────┬───────────┬──────────┬───────────┘
       │           │           │          │
┌──────▼───┐ ┌────▼─────┐ ┌───▼──────┐ ┌─▼──────────┐
│  用户服务 │ │ 商品服务  │ │ 订单服务  │ │  库存服务   │
│  :8081   │ │  :8082   │ │  :8083   │ │   :8084    │
└──────┬───┘ └────┬─────┘ └───┬──────┘ └─┬──────────┘
       │           │           │          │
┌──────▼───────────▼───────────▼──────────▼──────────┐
│                    MySQL 8.0                         │
│        user_db / product_db / order_db               │
└─────────────────────────────────────────────────────┘
                       ▲
┌──────────────────────┴──────────────────────────────┐
│                    Redis 7.x                         │
│           秒杀库存缓存 / Token缓存 / 限流              │
└─────────────────────────────────────────────────────┘
```

## 2. 服务拆分说明

### 2.1 用户服务（user-service: 8081）
**职责：** 用户生命周期管理

| 功能      | 说明                        |
|---------|-----------------------------|
| 用户注册  | 用户名/密码/邮箱注册，密码BCrypt加密 |
| 用户登录  | 验证凭据，返回 Token           |
| 用户信息  | 查询/更新用户基本信息            |

**数据库：** `user_db.t_user`

---

### 2.2 商品服务（product-service: 8082）
**职责：** 商品信息管理

| 功能      | 说明               |
|---------|------------------|
| 商品列表  | 分页查询商品信息      |
| 商品详情  | 根据ID查询商品       |
| 商品管理  | 上架/下架/修改商品    |

**数据库：** `product_db.t_product`

---

### 2.3 库存服务（inventory-service: 8084）
**职责：** 库存查询与扣减（秒杀核心）

| 功能      | 说明                              |
|---------|-----------------------------------|
| 查询库存  | 查询商品当前库存                    |
| 预扣库存  | 秒杀时 Redis 原子扣减，防止超卖      |
| 回滚库存  | 订单取消时归还库存                   |

**关键设计：** Redis `DECR` 原子操作保证库存扣减不超卖

**数据库：** `product_db.t_inventory`

---

### 2.4 订单服务（order-service: 8083）
**职责：** 订单创建与查询

| 功能      | 说明                  |
|---------|-----------------------|
| 创建订单  | 扣库存成功后异步创建订单 |
| 查询订单  | 用户订单列表/详情       |
| 订单状态  | 待支付/已支付/已取消    |

**数据库：** `order_db.t_order`

---

## 3. 秒杀核心流程

```
用户请求秒杀
    │
    ▼
Gateway 限流（令牌桶/Redis）
    │
    ▼
用户服务验证 Token
    │
    ▼
库存服务 Redis DECR（原子操作）
    ├── 库存不足 → 返回"秒杀结束"
    └── 扣减成功
            │
            ▼
        发送MQ消息（异步创建订单）
            │
            ▼
        订单服务消费消息，落库
            │
            ▼
        返回"秒杀成功，订单号: xxx"
```

## 4. 高并发三高设计

| 目标   | 实现手段                                     |
|------|------------------------------------------|
| 高并发  | Redis 原子操作扣库存，避免 MySQL 行锁竞争          |
| 高可用  | 各服务独立部署，单服务故障不影响整体（服务隔离）          |
| 高可靠  | 数据库事务 + 消息队列异步落单，保证最终一致性            |
